Code editor

Jtool now gives you the ability to edit the step event of many objects that you can place in the editor, giving them dynamic functionality. Now you can create moving obstacles, custom-angled spikes, and even complex triggers! Jtool uses the NSP code parser by Surgeon_, released under the GPLv3. This parser, while very powerful, is not perfect; see below for all limitations.

The following objects can have code added to them:
  • Spikes
  • Minispikes
  • Apples
  • Platforms
  • Blocks
  • Miniblocks
  • Triggers


The following functions are supported in code:
  • instance_create(x,y,obj)
  • instance_destroy()
  • instance_nearest(x,y,obj)
  • instance_exists(obj)
  • place_meeting(x,y,obj)
  • sin(rad)
  • cos(rad)
  • tan(rad)
  • arctan2(y,x)
  • degtorad(deg)
  • radtodeg(rad)
  • abs(val)
  • floor(val)
  • ceil(val)
  • round(val)
  • sqrt(val)
  • min(a,b)
  • max(a,b)
  • lengthdir_x(len,dir)
  • lengthdir_y(len,dir)
  • point_distance(x1,y1,x2,y2)
  • point_direction(x1,y1,x2,y2)
  • random(n)
  • irandom(n)
  • random_range(n1,n2)
  • irandom_range(n1,n2)


You may reference the following variables in your code:
  • User-Defined Instance Variables: a, b, c, d, i, t, z
  • GM-Defined Instance Variables: x, y, xstart, ystart, hspeed, vspeed, direction, image_angle, image_index, image_speed, sprite_index, object_index
  • Global Trigger Variable: global.trg


You can also use the following structures:
  • If statements: if (a == 1) { hspeed=-1 }
  • For & While loops
  • The With keyword: with(other) {...} with (all) {...}
